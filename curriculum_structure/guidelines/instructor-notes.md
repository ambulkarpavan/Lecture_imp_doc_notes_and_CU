## Overview

These guidelines are designed to help coding instructors create comprehensive, structured, and consistent notes for each concept covered in their sessions. These notes will serve as reference material for new instructors and ensure that the course material and learning objectives are aligned.

## Structure of Notes

Each set of notes should be organized around a specific concept or topic and follow this structure:

```markdown
# Concept X:

## Sample Lecture Link

- Provide a link to a previous session's recorded lecture (e.g., AWS S3 Link) for instructors to reference. This will help maintain consistency in teaching methodology.

## Introduction

- Offer a brief introduction to the concept or topic. Include its relevance to the course and its importance in the field of programming.

## Detailed Explanation

- **What is the concept?** - Define the concept and explain its fundamental principles.
- **Why is it useful?** - Discuss the use-case and benefits of the concept in real-world applications.
- **Real-world examples or analogies** - Provide examples or analogies to help learners understand the topic better.
- **Visual models** - If applicable, include diagrams, flowcharts, or other visual aids to enhance understanding.

## Instructor Activity | Code Implementation | Examples

- Detail the activities or demonstrations that the instructor will perform to illustrate the concept. This could include code snippets, algorithm walkthroughs, or live coding sessions.
- Provide explanations for each step of the code or activity to ensure clarity.

## Student Activities

- List exercises or assignments for students to practice and reinforce their understanding of the concept. Include a variety of tasks ranging from simple to complex.

## Conclusion

- Summarize the key points covered in the session. Reinforce the significance of the concept and its practical applications.

## Resources - Official Documentation and Other Resources

- Provide links to official documentation related to the concept ( must )
- Provide links to external resources such as documentation, tutorials, or articles for further reading. ( optional )
```

## Additional Tips

- **Consistency**: Ensure that the format and style of the notes are consistent across all concepts. All the concepts should have the following structure. If it is not applicable, please make sure that you still write the heading as asked and leave the content part empty.
- **Clarity**: Use simple language and clear explanations. Avoid jargon or overly technical terms without proper explanation.
- **Engagement**: Incorporate interactive elements such as quizzes, polls, or discussion prompts to engage students. That is highly recommended. All the quizzes wrt the session goes in learning-check file.
- **Feedback**: We do encourage feedback from everyone. If you have some feedback/suggestions. feel free to reach out to use and we definitely would love to hear.

- **Prompt** : Should you require assistance in generating content for a specific topic, the following GPT prompt may prove useful. Remember this does not have to be the exact prompt, Tweak it according to your needs.

```
Write a very deep, detailed and interactive lesson on the {topic} in around 4000 characters. Use relevant CODE EXAMPLES with explanation. Your explanations should be driven by code examples. keep it hands on.

Keep the following lesson structure:

## Introduction to { topic }
If possible, define {topic} in technical terms.

## Detailed Explanation
Explain {topic} to an absolute beginner. Use easy to understand Analogies and examples. Use step by step teaching. Start with simplest examples and build on top of that. Your explanations should be driven by code examples. keep it hands on.

## Use-case & benefits
Tell about the problems {topic} is solving. Give example of problem that {topic} solves and how it solves the problem.

## Real world Examples
Give examples that are easy to relate and relevant to the real world programming

## Usage
If relevant to {topic}, give all important methods, properties with definitions and example in a nice tabular format.

## Instructor Activity
A relatively big problem statement related to {topic} that instructor solves in class. Provide step by step explanation and the solution as well.

## Learners Activity
A relatively big problem statement related to {topic} that student solves in class. Provide step by step explanation and the solution as well.

## Learning Check
In the learning check give 3 Multiple choice questions about the topic and add 2 live coding challenges.

## Live Coding
Give 2 Live Coding Challenge
```

The whole objective of these guidelines is to help instructors create effective and valuable notes that will enhance the learning experience for students and provide a solid foundation for new instructors joining the organization.
